# gen_ppt

将结构化 Markdown 编译为 PPT（PowerPoint）演示文稿。

## 项目概述

本项目接受一份结构化的 Markdown 文件作为输入，解析其中的内容结构，自动生成对应的 PPT 页面。对 PPT 样式的要求极简，不追求复杂排版与动画效果，侧重于内容的清晰呈现。

## 页面类型

PPT 支持以下 4 种页面类型：

### 1. 封面页

- **内容**：PPT 标题 + 演讲人姓名
- **背景**：使用 `cover.png`（与 `build_ppt.py` 同目录）作为全幅背景图
- **后备**：若 `cover.png` 不存在，使用空白背景（纯白）

### 2. 内容页

- **内容**：由若干行文字组成，支持：
  - 有序列表（数字序号）
  - 无序列表（圆点/短横线序号）
  - Markdown 表格（表头 + 数据行），采用三段式样式（无底色填充、无竖线，仅顶线/中线/底线）
- **背景**：使用 `background.png`（与 `build_ppt.py` 同目录）作为全幅背景图
- **后备**：若 `background.png` 不存在，使用空白背景（纯白）
- **特殊布局**：若该页仅包含一个二级标题（`## `），无其他文字或表格，则该标题在幻灯片中水平和垂直居中显示

### 3. 图片展示页

- **内容**：
  - 顶部：图片说明文字（标题/描述）
  - 下方：展示图片，支持 1～6 张，自动选择布局
- **布局规则**：

| 图片数 | 布局 |
|--------|------|
| 1 | 居中大幅展示 |
| 2 | 左右均分 |
| 3 | 上排 2 张 + 下排 1 张居中 |
| 4 | 2×2 田字格 |
| 5～6 | 3 列网格（最多 2 行） |
| ≥7 | 仅取前 6 张并输出警告 |

- 每张图可附带 alt 文本（`![说明](path)`），渲染为图片下方的小号说明文字
- **背景**：使用 `background.png`（与 `build_ppt.py` 同目录）作为全幅背景图
- **后备**：若 `background.png` 不存在，使用空白背景（纯白）

### 4. 图配文页（尽可能少用）

- **内容**：左侧 2/3 为文字区域，右侧 1/3 为图片区域
  - 文字部分支持的数据类型和样式与"内容页"一致（二级标题、有序/无序列表、纯文本、Markdown 表格）
  - 图片部分展示一幅图片
- **检测规则**：页面同时包含结构化文字内容（二级标题、列表、表格等）和图片引用时，自动识别为图配文页
- **背景**：使用 `background.png`（与 `build_ppt.py` 同目录）作为全幅背景图
- **后备**：若 `background.png` 不存在，使用空白背景（纯白）

### 5. 结束页

- **内容**：「谢谢」（宋体 66pt，居中）
- **背景**：使用 `cover.png`（与 `build_ppt.py` 同目录）作为全幅背景图（与封面相同）
- **后备**：若 `cover.png` 不存在，使用空白背景（纯白）
- **说明**：无需在 Markdown 中编写，PPT 末尾自动生成

## 背景图片规则总结

| 页面类型   | 默认背景图片  | 缺失时回退 | 可通过元数据覆盖 |
| ---------- | ------------- | ---------- | ---------------- |
| 封面页     | `cover.png`   | 空白背景   | `<!-- cover: ... -->` |
| 内容页     | `background.png`   | 空白背景   | `<!-- background: ... -->` |
| 图片展示页 | `background.png`   | 空白背景   | `<!-- background: ... -->` |
| 图配文页   | `background.png`   | 空白背景   | `<!-- background: ... -->` |
| 结束页     | `cover.png`   | 空白背景   | `<!-- cover: ... -->` |

## 字体规范

| 文本类型   | 中文字体 | 英文字体          | 字号 |
| ---------- | -------- | ----------------- | ---- |
| 一级标题   | 宋体     | Times New Roman   | 66pt |
| 二级标题   | 宋体     | Times New Roman   | 44pt |
| 正文       | 仿宋     | Times New Roman   | 32pt |
| 表格表头   | 宋体     | Times New Roman   | 32pt |
| 表格内容   | 仿宋     | Times New Roman   | 32pt |
| 结束页     | 宋体     | Times New Roman   | 66pt |

- **一级标题**：Markdown 中的 `# ` 对应封面 PPT 标题
- **二级标题**：Markdown 中的 `## ` 对应内容页标题、演讲人姓名、图片说明文字
- **正文**：有序列表、无序列表、普通文本

## 输入格式

输入的 Markdown 文件（如 `content.md`）采用结构化语法来区分不同页面类型。

### 页面分隔

使用三个减号（`---`）作为页面分隔符。每一页由 `---` 开始（第一页可选），两个 `---` 之间的内容属于同一页 PPT。

### 元数据（可选）

在第一个 `---` 之前，可通过 HTML 注释指定背景图，覆盖默认值：

```markdown
<!-- cover: slides/cover.png -->
<!-- background: slides/bg.png -->

# 演示文稿标题
## 演讲人姓名
---
```

- `cover`：封面页及结束页的背景图
- `background`：内容页、图片展示页、图配文页的背景图
- 路径相对于 Markdown 文件所在目录；未指定时使用默认值（`cover.png` / `background.png`）

## 技术栈

- 语言：Python
- PPT 生成：`python-pptx` 库

## 文件结构

```
gen_ppt/
├── build_ppt.py       # 主构建脚本，读取 content.md 生成 .pptx
├── content.md         # 输入的结构化 Markdown 内容
├── cover.png          # 封面背景图（与 build_ppt.py 同目录）
├── background.png     # 内容页/图片展示页背景图（与 build_ppt.py 同目录）
├── src/               # 静态资源目录（图片等素材）
├── tmp/               # 临时资源目录
└── README.md          # 本文件
```
